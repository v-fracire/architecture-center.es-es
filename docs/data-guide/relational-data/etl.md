## <a name="extract-transform-and-load-etl"></a><span data-ttu-id="7c683-108">Extracción, transformación y carga de datos (ETL)</span><span class="sxs-lookup"><span data-stu-id="7c683-108">Extract, transform, and load (ETL)</span></span>

<span data-ttu-id="7c683-109">Extracción, transformación y carga (ETL) es una canalización de datos que se utiliza para recopilar datos de varios orígenes, transformar los datos según las reglas de negocio y cargarlos en un almacén de datos de destino.</span><span class="sxs-lookup"><span data-stu-id="7c683-109">Extract, transform, and load (ETL) is a data pipeline used to collect data from various sources, transform the data according to business rules, and load it into a destination data store.</span></span> <span data-ttu-id="7c683-110">El trabajo de transformación en ETL tiene lugar en un motor especializado y, a menudo, implica el uso de tablas de almacenamiento temporal para conservar los datos temporalmente a medida que estos se transforman y, finalmente, se cargan en su destino.</span><span class="sxs-lookup"><span data-stu-id="7c683-110">The transformation work in ETL takes place in a specialized engine, and often involves using staging tables to temporarily hold data as it is being transformed and ultimately loaded to its destination.</span></span>

<span data-ttu-id="7c683-111">La transformación de datos que tiene lugar a menudo conlleva varias operaciones como filtrado, ordenación, agregación, combinación de datos, limpieza de datos, desduplicación y validación de datos.</span><span class="sxs-lookup"><span data-stu-id="7c683-111">The data transformation that takes place usually involves various operations, such as filtering, sorting, aggregating, joining data, cleaning data, deduplicating, and validating data.</span></span>

## <a name="extract-load-and-transform-elt"></a><span data-ttu-id="7c683-119">Extracción, carga y transformación (ELT)</span><span class="sxs-lookup"><span data-stu-id="7c683-119">Extract, load, and transform (ELT)</span></span>

<span data-ttu-id="7c683-120">Extracción, carga y transformación (ELT) difiere de ETL solo por la ubicación en la que se realiza la transformación.</span><span class="sxs-lookup"><span data-stu-id="7c683-120">Extract, load, and transform (ELT) differs from ETL solely in where the transformation takes place.</span></span> <span data-ttu-id="7c683-121">En la canalización de ELT, la transformación se produce en el almacén de datos de destino.</span><span class="sxs-lookup"><span data-stu-id="7c683-121">In the ELT pipeline, the transformation occurs in the target data store.</span></span> <span data-ttu-id="7c683-122">En lugar de usar un motor de transformación independiente, las funcionalidades de procesamiento del almacén de datos de destino se utilizan para transformar los datos.</span><span class="sxs-lookup"><span data-stu-id="7c683-122">Instead of using a separate transformation engine, the processing capabilities of the target data store are used to transform data.</span></span> <span data-ttu-id="7c683-123">Esto simplifica la arquitectura ya que permite quitar el motor de transformación de la canalización.</span><span class="sxs-lookup"><span data-stu-id="7c683-123">This simplifies the architecture by removing the transformation engine from the pipeline.</span></span> <span data-ttu-id="7c683-124">Otra ventaja de este enfoque es que al escalar el almacén de datos de destino también se escala el rendimiento de la canalización de ELT.</span><span class="sxs-lookup"><span data-stu-id="7c683-124">Another benefit to this approach is that scaling the target data store also scales the ELT pipeline performance.</span></span> <span data-ttu-id="7c683-125">No obstante, ELT solo funciona bien si el sistema de destino tiene la suficiente potencia para transformar los datos de forma eficaz.</span><span class="sxs-lookup"><span data-stu-id="7c683-125">However, ELT only works well when the target system is powerful enough to transform the data efficiently.</span></span>

![Proceso de extracción, carga y transformación (ELT)](../images/elt.png)

<span data-ttu-id="7c683-127">Los casos de uso habituales para ELT están dentro del dominio de los macrodatos.</span><span class="sxs-lookup"><span data-stu-id="7c683-127">Typical use cases for ELT fall within the big data realm.</span></span> <span data-ttu-id="7c683-128">Por ejemplo, puede comenzar extrayendo todos los datos de origen a archivos planos en un almacenamiento escalable como el sistema de archivos distribuido de Hadoop (HDFS) o Azure Data Lake Store.</span><span class="sxs-lookup"><span data-stu-id="7c683-128">For example, you might start by extracting all of the source data to flat files in scalable storage such as Hadoop distributed file system (HDFS) or Azure Data Lake Store.</span></span> <span data-ttu-id="7c683-129">Se pueden utilizar tecnologías como Spark, Hive o PolyBase para consultar los datos de origen.</span><span class="sxs-lookup"><span data-stu-id="7c683-129">Technologies such as Spark, Hive, or PolyBase can then be used to query the source data.</span></span> <span data-ttu-id="7c683-130">El punto clave de ELT es que el almacén de datos que se usa para realizar la transformación es el mismo almacén de datos en el que se consumen en última instancia los datos.</span><span class="sxs-lookup"><span data-stu-id="7c683-130">The key point with ELT is that the data store used to perform the transformation is the same data store where the data is ultimately consumed.</span></span> <span data-ttu-id="7c683-131">Este almacén de datos lee directamente desde el almacenamiento escalable, en lugar de cargar los datos en su propio almacenamiento propietario.</span><span class="sxs-lookup"><span data-stu-id="7c683-131">This data store reads directly from the scalable storage, instead of loading the data into its own proprietary storage.</span></span> <span data-ttu-id="7c683-132">Este enfoque omite el paso de copia de datos que se da en ETL. Esta es una operación que lleva mucho tiempo en el caso de conjuntos de datos grandes.</span><span class="sxs-lookup"><span data-stu-id="7c683-132">This approach skips the data copy step present in ETL, which can be a time consuming operation for large data sets.</span></span>

<span data-ttu-id="7c683-133">En la práctica, el almacén de datos de destino es un [almacenamiento de datos](./data-warehousing.md) que usa un clúster de Hadoop (con Hive o Spark) o una instancia de SQL Data Warehouse.</span><span class="sxs-lookup"><span data-stu-id="7c683-133">In practice, the target data store is a [data warehouse](./data-warehousing.md) using either a Hadoop cluster (using Hive or Spark) or a SQL Data Warehouse.</span></span> <span data-ttu-id="7c683-134">En general, un esquema se superpone sobre los datos del archivo plano en el momento de la consulta y se almacena como una tabla, lo cual permite que se puedan realizar consultas a los datos como en cualquier otra tabla del almacén de datos.</span><span class="sxs-lookup"><span data-stu-id="7c683-134">In general, a schema is overlaid on the flat file data at query time and stored as a table, enabling the data to be queried like any other table in the data store.</span></span> <span data-ttu-id="7c683-135">Estas se conocen como tablas externas porque los datos no residen en el almacenamiento administrado por el almacén de datos en sí mismo, sino en algunos almacenamientos escalables externos.</span><span class="sxs-lookup"><span data-stu-id="7c683-135">These are referred to as external tables because the data does not reside in storage managed by the data store itself, but on some external scalable storage.</span></span> 

<span data-ttu-id="7c683-136">El almacén de datos solo administra el esquema de los datos y aplica el esquema en la lectura.</span><span class="sxs-lookup"><span data-stu-id="7c683-136">The data store only manages the schema of the data and applies the schema on read.</span></span> <span data-ttu-id="7c683-137">Por ejemplo, un clúster de Hadoop que use Hive describiría una tabla de Hive en la que el origen de datos es, efectivamente, una ruta de acceso a un conjunto de archivos en HDFS.</span><span class="sxs-lookup"><span data-stu-id="7c683-137">For example, a Hadoop cluster using Hive would describe a Hive table where the data source is effectively a path to a set of files in HDFS.</span></span> <span data-ttu-id="7c683-138">En SQL Data Warehouse, PolyBase puede lograr el mismo resultado &mdash; creando una tabla en los datos almacenados de forma externa a la propia base de datos.</span><span class="sxs-lookup"><span data-stu-id="7c683-138">In SQL Data Warehouse, PolyBase can achieve the same result &mdash; creating a table against data stored externally to the database itself.</span></span> <span data-ttu-id="7c683-139">Una vez cargados los datos de origen, los datos presentes en las tablas externas se pueden procesar mediante las funcionalidades del almacén de datos.</span><span class="sxs-lookup"><span data-stu-id="7c683-139">Once the source data is loaded, the data present in the external tables can be processed using the capabilities of the data store.</span></span> <span data-ttu-id="7c683-140">En escenarios de macrodatos, esto significa que el almacén de datos debe ser capaz de realizar un procesamiento paralelo masivo (MPP) que divide los datos en fragmentos más pequeños y distribuye el procesamiento de los fragmentos en varias máquinas en paralelo.</span><span class="sxs-lookup"><span data-stu-id="7c683-140">In big data scenarios, this means the data store must be capable of massively parallel processing (MPP), which breaks the data into smaller chunks and distributes processing of the chunks across multiple machines in parallel.</span></span>

<span data-ttu-id="7c683-141">La fase final de la canalización de ELT suele transformar los datos de origen a un formato final que es más eficaz para los tipos de consultas que se deben admitir.</span><span class="sxs-lookup"><span data-stu-id="7c683-141">The final phase of the ELT pipeline is typically to transform the source data into a final format that is more efficient for the types of queries that need to be supported.</span></span> <span data-ttu-id="7c683-142">Por ejemplo, se pueden realizar particiones en los datos.</span><span class="sxs-lookup"><span data-stu-id="7c683-142">For example, the data may be partitioned.</span></span> <span data-ttu-id="7c683-143">Además, ELT podría usar formatos de almacenamiento optimizado como Parquet, que almacena datos orientados a filas en forma de columnas y proporciona una indexación optimizada.</span><span class="sxs-lookup"><span data-stu-id="7c683-143">Also, ELT might use optimized storage formats like Parquet, which stores row-oriented data in a columnar fashion and providess optimized indexing.</span></span>
