## <a name="distributedtokencache-implementation"></a><span data-ttu-id="0e998-136">Implementación de DistributedTokenCache</span><span class="sxs-lookup"><span data-stu-id="0e998-136">DistributedTokenCache implementation</span></span>
<span data-ttu-id="0e998-137">La clase `DistributedTokenCache` se deriva de la clase [TokenCache][tokencache-class] de la biblioteca de autenticación de Azure AD.</span><span class="sxs-lookup"><span data-stu-id="0e998-137">The `DistributedTokenCache` class derives from the ADAL [TokenCache][tokencache-class] class.</span></span>

<span data-ttu-id="0e998-138">En el constructor, la clase `DistributedTokenCache` crea una clave para el usuario actual y la carga la memoria caché desde la memoria auxiliar:</span><span class="sxs-lookup"><span data-stu-id="0e998-138">In the constructor, the `DistributedTokenCache` class creates a key for the current user and loads the cache from the backing store:</span></span>

```csharp
public DistributedTokenCache(
    ClaimsPrincipal claimsPrincipal,
    IDistributedCache distributedCache,
    ILoggerFactory loggerFactory,
    IDataProtectionProvider dataProtectionProvider)
    : base()
{
    _claimsPrincipal = claimsPrincipal;
    _cacheKey = BuildCacheKey(_claimsPrincipal);
    _distributedCache = distributedCache;
    _logger = loggerFactory.CreateLogger<DistributedTokenCache>();
    _protector = dataProtectionProvider.CreateProtector(typeof(DistributedTokenCache).FullName);
    AfterAccess = AfterAccessNotification;
    LoadFromCache();
}
```

<span data-ttu-id="0e998-139">La clave se crea concatenando el identificador de usuario y el identificador de cliente.</span><span class="sxs-lookup"><span data-stu-id="0e998-139">The key is created by concatenating the user ID and client ID.</span></span> <span data-ttu-id="0e998-140">Ambos se toman de las notificaciones encontradas en `ClaimsPrincipal`del usuario:</span><span class="sxs-lookup"><span data-stu-id="0e998-140">Both of these are taken from claims found in the user's `ClaimsPrincipal`:</span></span>

```csharp
private static string BuildCacheKey(ClaimsPrincipal claimsPrincipal)
{
    string clientId = claimsPrincipal.FindFirstValue("aud", true);
    return string.Format(
        "UserId:{0}::ClientId:{1}",
        claimsPrincipal.GetObjectIdentifierValue(),
        clientId);
}
```

<span data-ttu-id="0e998-141">Para cargar los datos de la memoria caché, lea el blob serializado desde la memoria auxiliar y llame a `TokenCache.Deserialize` para convertir el blob en datos de memoria caché.</span><span class="sxs-lookup"><span data-stu-id="0e998-141">To load the cache data, read the serialized blob from the backing store, and call `TokenCache.Deserialize` to convert the blob into cache data.</span></span>

```csharp
private void LoadFromCache()
{
    byte[] cacheData = _distributedCache.Get(_cacheKey);
    if (cacheData != null)
    {
        this.Deserialize(_protector.Unprotect(cacheData));
    }
}
```

<span data-ttu-id="0e998-142">Cada vez que ADAL acceda a la memoria caché, se activará un evento `AfterAccess` .</span><span class="sxs-lookup"><span data-stu-id="0e998-142">Whenever ADAL access the cache, it fires an `AfterAccess` event.</span></span> <span data-ttu-id="0e998-143">Si han cambiado los datos de la memoria caché, la propiedad `HasStateChanged` es true.</span><span class="sxs-lookup"><span data-stu-id="0e998-143">If the cache data has changed, the `HasStateChanged` property is true.</span></span> <span data-ttu-id="0e998-144">En ese caso, actualice la memoria auxiliar para reflejar el cambio y, a continuación, establezca `HasStateChanged` en false.</span><span class="sxs-lookup"><span data-stu-id="0e998-144">In that case, update the backing store to reflect the change, and then set `HasStateChanged` to false.</span></span>

```csharp
public void AfterAccessNotification(TokenCacheNotificationArgs args)
{
    if (this.HasStateChanged)
    {
        try
        {
            if (this.Count > 0)
            {
                _distributedCache.Set(_cacheKey, _protector.Protect(this.Serialize()));
            }
            else
            {
                // There are no tokens for this user/client, so remove the item from the cache.
                _distributedCache.Remove(_cacheKey);
            }
            this.HasStateChanged = false;
        }
        catch (Exception exp)
        {
            _logger.WriteToCacheFailed(exp);
            throw;
        }
    }
}
```

<span data-ttu-id="0e998-145">TokenCache envía otros dos eventos:</span><span class="sxs-lookup"><span data-stu-id="0e998-145">TokenCache sends two other events:</span></span>

* <span data-ttu-id="0e998-146">`BeforeWrite`.</span><span class="sxs-lookup"><span data-stu-id="0e998-146">`BeforeWrite`.</span></span> <span data-ttu-id="0e998-147">Se le llama inmediatamente antes de que ADAL escriba en la memoria caché.</span><span class="sxs-lookup"><span data-stu-id="0e998-147">Called immediately before ADAL writes to the cache.</span></span> <span data-ttu-id="0e998-148">Puede utilizar esto para implementar una estrategia de simultaneidad</span><span class="sxs-lookup"><span data-stu-id="0e998-148">You can use this to implement a concurrency strategy</span></span>
* <span data-ttu-id="0e998-149">`BeforeAccess`.</span><span class="sxs-lookup"><span data-stu-id="0e998-149">`BeforeAccess`.</span></span> <span data-ttu-id="0e998-150">Se le llama inmediatamente antes de que ADAL lea desde la memoria caché.</span><span class="sxs-lookup"><span data-stu-id="0e998-150">Called immediately before ADAL reads from the cache.</span></span> <span data-ttu-id="0e998-151">Aquí puede volver a cargar la memoria caché para obtener la versión más reciente.</span><span class="sxs-lookup"><span data-stu-id="0e998-151">Here you can reload the cache to get the latest version.</span></span>

<span data-ttu-id="0e998-152">En nuestro caso, decidimos no administrar estos dos eventos.</span><span class="sxs-lookup"><span data-stu-id="0e998-152">In our case, we decided not to handle these two events.</span></span>

* <span data-ttu-id="0e998-153">Para la simultaneidad, la última escritura tiene prioridad.</span><span class="sxs-lookup"><span data-stu-id="0e998-153">For concurrency, last write wins.</span></span> <span data-ttu-id="0e998-154">Eso está bien, porque los tokens se almacenan por separado para cada usuario + cliente, por lo que solo podría ocurrir un conflicto si el mismo usuario tuviera dos inicios de sesión abiertos simultáneamente.</span><span class="sxs-lookup"><span data-stu-id="0e998-154">That's OK, because tokens are stored independently for each user + client, so a conflict would only happen if the same user had two concurrent login sessions.</span></span>
